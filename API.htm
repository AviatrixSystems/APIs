1         Login

1.1       Description

Login to controller with valid username and password.

1.2       Request Parameters

action

Indicating type of API to call.

Type: String

Valid value: "login"

Default: None

Required: Yes

username

The user name for the account.

Type: String

Valid Value: "admin" or an existing user account name

Default: None

Required: Yes

password

The password for the account user. For security and usability, we recommend you use the password in POST request. Using password in GET would require you to url encode any special characters in password.

Type: String

Default: None

Required: Yes

 

1.3       Response Elements

return

API return

Type: String ("true" or "false")

Valid Values: true | false

CID

API session ID

Type: String

reason

Reason of failed request if return is "false".

Type: String

Valid Values: reason is empty string if return is "true"

results

results if return is true

Type: String

Valid Values: result is empty string if return is "false"

 

1.4       Example Request 1 (Recommended way)

Available in 3.0 or later release.

curl -k --data "password=Aviatrix123!" "https://10.123.123.123/v1/api?action=login&username=admin"

1.4       Example Request 2 (Deprecated).

To use in version 2.7 or earlier version.

https://192.168.0.251/v1/api?action=login&username=admin&password=password

33    Upgrade

 

33.1  Description

Upgrade the controller to a specific release or to the latest.

33.2  Request Parameters

action

Indicating type of API to call.

Type: String

Valid value: "upgrade"

Default: None

Required: Yes

CID

API session ID

Type: String

Default: None

Required: Yes

version

The release version number to which the controller will be upgraded to. If not specified, it automatically will be upgraded to the latest release.

Type: String

Default: None

Required: No

33.3  Response Elements

return

API return

Type: String ("true" or "false")

Valid Values: true | false

reason

Reason of failed request if return is "false".

Type: String

Valid Values: reason is empty string if return is "true"

results

results if return is true

Type: String

Valid Values: result is empty string if return is "false"

 

33.4  Example Request

https://192.168.0.251/v1/api?CID=54557b087a48a&action=upgrade&version=031717

33.5  Example Response

{

  "return": true,

  "results": "userConnect has been upgraded to version UserConnect-031717.152. Please log out and login again for the new changes to take effect."

}
